## Severity Definitions

| Severity | Definition |
|----------|------------|
| **Blocker** | Feature completely broken, data loss risk, security issue, crashes the app |
| **High** | Core feature works but significant issue, workaround exists but is poor UX |
| **Medium** | Non-critical feature issue, poor UX, workaround easy |
| **Low** | Cosmetic, minor UX annoyance, no functional impact |
| **Won't Fix** | Won't address in this sprint/launch cycle |
